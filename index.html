一、产品概述

1.1 产品定位

Maygrove 是一款面向家庭园艺爱好者的植物种植伴侣应用，以 persona「Mavi」陪伴用户完成植物从种子到餐桌的全过程。目前 App 已具备 Smart Care、Thrive、Eco、Vacation 等核心种植模式。

本次建设两大核心板块：

  1. 订阅会员体系（Free / Basic / Pro 三档）
  2. 种子商城（从 0 到 1 自建供应链电商能力）

1.2 核心原则

原则说明
分阶段上线v1（2026.10）：Free + Basic（含种子商城从 0 到 1）→ v2：Pro 高级会员
Basic = 供应链权益 bundleBasic 仅包含种子/耗材/物流等电商权益，不解锁任何软件功能
Pro = 电商权益 + 软件功能解锁Pro 在 Basic 的基础上叠加 AI 助手无限制、延时摄影、AI 记忆等软件权益
护城河功能保持免费假期模式、快速生长、节能、自动换水、自动报警 — 对所有用户免费开放
种子积分体系引入积分作为会员权益的载体，支持按月发放和发芽保障赔付

四、关键用户流程

4.1 主流程 A：Free → Basic 升级 → 首单种子

Free 用户 → 进入种子商店 → 浏览种子 → 看到会员价锁定/原价 → 点击「升级 Basic 享 6 折+免费积分+包邮」 → 进入会员中心 → 权益对比表（Free vs Basic） → 选择升级 → 确认支付（$19.99~24.99/月） → 支付成功 → ├─ 等级立即升级为 Basic ├─ 当月种子积分到账 └─ 跳转种子商店 → 种子全部显示 6 折会员价 + 积分兑换标签 → 选购种子（支持 1 颗起买+混搭）→ 加购 → 结算时自动应用： ├─ 种子 6 折 ├─ 积分抵扣（可选） ├─ 首次包邮 └─ 生日礼券（如当月生日） → 下单成功

4.2 主流程 B：发芽保障申请

Basic 用户 → 购买的种子播种后超过约定天数未发芽 → 在订单详情页 → 发起「发芽保障」 → 填写保障申请： ├─ 选择种子品种 ├─ 上传未发芽照片 └─ 提交 → 后台审核（48h 内）： ├─ 通过 → 赔偿积分自动到账 └─ 不通过 → 告知原因 → 用户收到通知：「🌱 发芽保障审核通过，X 积分已到账」

4.3 分支流程

分支条件流程说明
Free 用户直接买种子只能购买预定套装/起订量，不能单颗买，不能混搭，无折扣
Basic 用户每月第二次下单需付运费（首次已免），其他折扣正常享受
Basic 用户取消订阅权益持续到周期结束，到期降级后所有电商权益失效
Basic 年付用户中途取消不按比例退款，权益持续到周期结束
积分过期过期前 7 天推送提醒：「你的种子积分即将过期，快去兑换～」
Free 用户收到生日礼券不适用（生日礼券仅 Basic+）
用户不填生日生日礼券不触发，会员页面可提示「填写生日领取专属礼券」

4.4 异常流程

异常场景系统处理用户看到
IAP 支付回调延迟客户端轮询最多 30s，超时标记待确认「支付处理中，请稍候…」
积分发放失败（定时任务异常）标记失败记录，重试 3 次，超时告警用户无感知
折扣计算错误（配置异常）回退原价，记录错误日志仅显示原价，不展示错误
发芽保障申请（种植后超过 30 天）系统校验天数，超 30 天自动驳回「该种子已超过发芽保障期限」
发芽保障照片疑似造假人工审核标记「审核中，我们会在 48h 内回复」
包邮次数跨月异常（月末下单发货跨月）以下单时间判定，非发货时间正常展示
用户生日当天支付成功立即发放当月积分 + 生日礼券「欢迎成为 Basic 会员！🎂 生日礼券已送达」

七、数据实体（轻量级）

实体业务含义关系
